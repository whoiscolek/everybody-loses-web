# Everybody Loses v10.83 Release Report

## Purpose

v10.83 fixes the reliability issues reported after v10.82:

- Old unmatched/open bets could remain visible under **My Bets** after the event was final, stale, archived, or missing from the current event cache.
- Net settle-up could leave a fake remainder when two users had open ledger rows in both directions.
- The Admin complete ledger summary undercounted matches because it only counted documents still present in the `matches` collection, even though older settled decisions can survive only as `ledgerEntries` after cleanup.
- Admins needed an in-app tool to grant admin privileges to another user.

## Changes

### My Bets stale record cleanup

- Client-side **My Bets** now treats stale missing-event records as not current after a 36-hour orphan grace window.
- Client-side **My Bets** hides open or matched records once their linked event is final, archived, hidden from Now, or older than the maximum live window.
- Server maintenance now expires stale open bets by setting their status to `expired` with a `staleReason`.
- Maintenance summary now reports `staleOpenBetsExpired`.

### Net settlement fix

- Settle-up now recalculates the current net balance from open ledger rows instead of trusting the displayed amount.
- When someone marks a balance settled, all open ledger rows between those two users are marked settled, including rows in both directions.
- The settlement record stores the recalculated net amount, included ledger row count, displayed amount, and whether the displayed amount differed from the recalculated total.
- This prevents a leftover balance like `$3` after a net payment already settled the whole relationship.

### Admin ledger match count fix

- The Admin complete ledger now counts matched decisions from both live `matches` docs and ledger-only archived decisions.
- Ledger-only decisions are displayed as archived ledger rows, so old settled decisions are still auditable even after old bet/match docs are cleaned up.
- The count labels were clarified to distinguish active bet rows, matched decisions, ledger-only decisions, ledger rows, and payments.

### Admin privilege manager

- Added an Admin-only **Admin privilege manager** card.
- Admins can select an existing user and grant admin privileges.
- Granting admin also approves the user.
- Admins can revoke another user’s admin privileges while keeping them approved.
- The UI blocks revoking admin from the currently signed-in admin account to reduce lockout risk.

## Firestore rules

No Firestore rules change is required if the v10.80+ rules are already deployed. Existing rules already allow approved admins to update user documents while preventing normal users from approving or promoting themselves.

## Deployment check

After deployment:

1. Open `/api/health` and confirm version `10.83`.
2. Open Admin and run **Run server refresh now** once.
3. Check the maintenance result for `staleOpenBetsExpired`.
4. Check **My Bets** for stale baseball/UFC records.
5. Check Ledger between you and Jamie; settle-up should clear the full net relationship instead of leaving a fake remainder.
6. Check Admin → Complete betting ledger; the match count should now include ledger-backed archived decisions.
7. Check Admin → Admin privilege manager for the new grant/revoke controls.
