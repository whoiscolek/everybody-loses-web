# Everybody Loses v10.80 Release Report

## Production issue addressed

The public health route worked on v10.79, but every Firebase-backed Vercel function returned `FUNCTION_INVOCATION_FAILED`. Without the Vercel invocation logs, the exact platform exception could not be proven. v10.80 removes the most fragile startup condition: Firebase Admin is no longer imported at module evaluation time.

## Reliability changes

- Firebase Admin modules load lazily inside a guarded request path.
- SDK import, credential parsing, initialization, authentication, and Firestore read failures now have separate structured error codes/stages.
- Firebase Admin is pinned to 13.10.0, a stable Node 22-compatible release, instead of the newly released 14.x major version.
- `/api/admin-health` verifies the Firebase-backed server independently before a multi-stage maintenance run.
- Manual maintenance stops launching repeated server calls when preflight fails and immediately runs the browser recovery path.
- The Admin page contains a read-only complete betting ledger covering every bet, match relationship, ledger result, and payment.

## Release gates

- Firebase-backed endpoints return structured configuration failures in local no-credential tests instead of crashing

## Deployment verification

After deployment:

1. `/api/health` must report version `10.80` and Node 22.
2. Sign in as admin and press **Run server refresh now**.
3. The first visible stage is **Checking Firebase server access…**.
4. If Firebase credentials are malformed, the Admin page now displays the exact stage and code instead of launching five failing requests.
5. When preflight succeeds, refresh, discovery, and settlement proceed normally.
