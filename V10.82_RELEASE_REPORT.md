# Everybody Loses v10.82 Release Report

## Reason for release

Profile stats were correctly computing settled wins/losses, but some older settled decisions could show as `Unknown Sport` / `Unknown league`. The likely production path was that history cleanup or old data migrations left `ledgerEntries` intact while removing or omitting the linked `events`, `bets`, or `matches` records that held sport and league identity.

## Fix

v10.82 makes sport/league identity durable and recoverable:

- Profile analytics now resolves sport and league from the live event first, then ledger fields, event snapshots, match/bet snapshots, event IDs, short codes, notes, and known NBA team text.
- Legacy NBA ledger rows can now be grouped as Basketball / NBA even when the original event document is gone.
- Newly written ledger rows now include compact event identity fields: `eventTitle`, `eventSport`, `eventLeague`, `eventShortCode`, and `eventSnapshot`.
- Server maintenance history cleanup now stamps surviving ledger rows with event identity before deleting old event/bet/match documents.
- Client-side manual settlement and ranked/UFC/team settlement paths also write the same event identity snapshot.

## Expected result after deploy

For the data pattern shown by the user, the prior 8-7 `Unknown Sport` / `Unknown league` bucket should collapse into Basketball / NBA when the ledger rows contain NBA-like IDs, short codes, or NBA team names in their notes/title/snapshots.

If any unknown rows remain after v10.82, that means those specific rows no longer contain any recoverable NBA identifier, team names, short code, league, or event snapshot. Future rows are protected by the new snapshot writes.
